1000-Introduction


INTRODUCTION TO SYMCALC

SymCalc is the SymbOS based spreadsheet program with powerful calculating abilities.

You can use SymCalc worksheets for accounting applications to keep track of profit and loss in a business, or create a worksheet that will calculate and display a schedule of mortgage payments for your home. SymCalc handles these and other calculating tasks with ease and reliability. And because SymCalc is a SymbOS application you will be using the same easy to learn and use graphic environment that you are already familiar with.

Unlike some other spreadsheet programs you may have used, SymCalc does not require that you learn a lot of complicated, or illogical commands in order to create and start working with your worksheet. Once you start up SymCalc you can immediately begin to type data into a worksheet that looks like a worksheet.

3100-Screen Layout


SCREEN LAYOUT

Before you start working with SymCalc you should have a quick overview of the screen components.


Menu bar

Like many SymbOS applications SymCalc contains a menu bar at the top of the window, were you reach all available functions.

Tool bar

Below the menu bar you will find the tool bar which contains icons for the most used functions.
- New, Open, Save, Print: See "File" menu
- Bold, Italic, Left aligned, Centered, Right aligned, Colours: See "Format" menu

Cell address field

The cell address field is placed on the left side below the tool bar and shows the address of the actual cell or the selected cell range. It can also be used for typing in another cell for placing the cursor or a cell range for selecting an amount of cells. See The Spreadsheet.

Cell editor

The cell editor is placed on the right side below the tool bar. It shows the content of the actual cell and allows to edit it. See Editing Cells.

Spreadsheet, column and row bars

The spreadsheet is using the largest part of the screen and shows the current document with its cell matrix. See The Spreadsheet.

Status bar

To gain more space on the screen, both the tool bar and the status bar can be turned off, see "View" menu.

3200-The Spreadsheet


THE SPREADSHEET

The spreadsheet is the document, with which are are working in SymCalc. It allows you to arrange different kinds of text and numeric data in a clear way and do powerful calculations with the data very easily. You can keep track of profit and loss in your big business, check your monthly private costs, do some funny math stuff or just write down a pizza order list.

Cell Matrix

A spreadsheet consists of a matrix of cells. Each cell may contain a text, a numeric value, a formula with a numeric result, or just nothing. Cells are arranged by columns and rows. Columns can be identified by the letters at the upper bar, rows by the numbers at the left bar. Each cell is referenced by its column and row, so e.g. the second cell in the forth row is B4, while the third cell in the first row is C1 etc.

How To Navigate

(check the Keyboard Shortcuts as well)

To navigate through a spreadsheet you can either use the mouse by clicking cells and scrollbars or use the cursor keys. The identifier of the actual cell is displayed in the upper left field above the spreadsheet. Its full content is displayed in the cell editor field on the upper right side. Both the mouse wheel can be used to scroll through the spreadsheet as well as cursor and control to skip pagewise. On advanced keyboards you can use page up/down or home/end for navigation. Another possibility is to type-in the cell address (e.g. C3) directly in the cell address field, and the cursor will jump to this one when pressing [Return].

Selecting Cell Ranges

SymCalc provides many operations which can be applied to multiple cells, so you may want to select more than one cell at the same time. Navigate to the first cell, then use the cursor keys together with shift to mark the desired cell range. You can use the mouse as well, click on the first cell, keep the left mouse button down and move over the desired range to mark all cells. An additional possibility is to type-in the cell range in the cell address field and press [Return]. The first and the last cell have to be specified, separated by a colon (e.g. A1:D4).

Editing Cells

To edit a cell, you can either just double click it, click the editor, press F2 or you just enter chars after selecting a cell. Please see the next chapter Editing Cells for detailed information about how to edit a cell.

Context Menus

In SymCalc you can open dedicated context menus by right-clicking the mouse over different areas.

For inserting, removing or modifying rows or columns, you have to right-click on the left or upper bar. Here you can remove or insert columns and rows. You can also change the width of columns or the height of rows. By selecting multiple columns or rows you can change all their sizes in one step.

Right-clicking on an actual cell or a selected cell range will open a context menu with a lot of options what to do now. For detailed information about cut, copy and paste please read the description of the Edit Menu. For detailed information about inserting and removing cells, rows and columns please read the description of the Table Menu.

Row And Column Sizes

SymCalc also allows to change the width of a column or the height of a row directly by using the powerful GUI capabilities of the SymbOS operating system. Just click on a separation line inside the column or the row bar: A dotted line will appear any you can move it to change the with of a column or the height of a row very easy.

3300-Editing Cells


EDITING CELLS

To enter new content for a cell, select it and just start typing. This will overwrite the previous content. If you want to edit an existing content of a cell you can either just double click it, click the editor or press F2. In both cases you will see and edit the content in the cell editor at the upper right screen.

Cell Types

Beside of beeing empty cells can contain three different kinds of content:
→ Text: You can enter any kind of text. This is for labeling rows and columns or just for adding descriptions in your spreadsheet. Cells with texts can't be used for calculations and will be ignored in several functions like the statistical ones.
→ Numeric value: You can enter different kinds of numeric data. These can be ordinal decimal numbers, numbers using the scientific notation or even dates, times, datetimes, binary or hexadecimal expressions. See here for detailed information about how to enter different kind of values.
→ Formulas: Any formula is starting with an equality sign (=). After this a formula is written in the same way like you know it from todays standards in math and programming languages. You can use operands like values and cell references, operators like * and + for calculation, parenthesis for changing the order of operations, and of course all the many functions provided by SymCalc. See here for detailed information about how to enter a formula.

Entering A New Cell

As soon as you are finished with entering the cell content you can just press [Return] or [Tab]. This will save the new cell content. If you don't want the new content but keep the old one, just press [Esc]. In both cases you will return back to the spreadsheet.

If SymCalc can't detect a numeric value or a formula at all it will always handle the input as a text cell. So if you type in a number like 5643890563728952, which is too large for beeing stored as a numeric value, SymCalc will just keep it as text.

If you entered a new cell, which was empty before, SymCalc tries to format the cell with the new cell type. That means, that binary, hexadecimal, date, time and datetime cells will always be displayed using the standard format of the related type.

Using the mouse for cell references

When editing a formula, you can use the mouse to insert references to individual cells or cell ranges. To do this, simply click on the cell in the spreadsheet or select a cell range. Its references are then automatically inserted into the cell content.

Recalculation

Once the value of a cell has changed, SymCalc checks to see if other cells reference the updated cell. If so, those cells are recalculated and all cells that reference them are also recalculated. It doesn't matter what order the cells are in the table, SymCalc will find all references in any direction. Although this uses a particularly fast algorithm, this process can still take a while for large, more complex tables.

3400-Number Input Formats


NUMBER INPUT FORMATS

SymCalc supports different types of number formats when you enter a value for a cell or within a formula. The most common formats are regular numbers like 262 or -144.256. By using special prefixes you can also enter and edit other types of numeric data.

Overview

Type Prefix Example
decimal [none] -123,456
scientific [none] 1,23e-23
percentage [none] 95%
binary % %1100101
hexadecimal # #4BF80
date @ @2024.08.13
time _ _17:59:32
datetime @..._ @2024.08.13_17:59:32

Limitations

For storing and calculating numeric values SymCalc is using the CPCs original powerful 40bit floating point engine, which is binary based and uses a 32bit mantissa and an 8bit exponent. It already allows better high precision math operations compared to many other floating point engines from the 1980ies and 1990ies, which were often only 32bit and/or decimal based.

Number are limited to a minimum/maximum decimal exponent of -39/+38. Binary, hexadecimal and datetime values are using the mantissa part only, which allows to handle 32bit numbers and any dates between 1980 and 2047.

Decimal

A decimal number has to start with a digit (0-9) or a minus (-) followed by an amount of additional digits. It may contain a decimal point, which is a point (.) or a comma (,) depending on the regional settings. It must not contain any 1000 separators.

Example: -123,456
Limits: 9 digits

Scientific

A number in scientific notation is a decimal number followed by an "e" which adds the base-ten exponent. The exponent may start with a minus (-) but must not contain a decimal point.

Example: 1,23e-23
Limits: 3e-39 -> 1,7e+38

Percentage

Percentage values can be entered like decimal numbers directly folowed by a "%" char. SymCalc will then convert them into ordinal decimal numbers just by dividing them by 100.

Example: 95% (will be converted to 0,95)

Binary

Binary numbers are identified with a "%" char as the prefix. They can only contain 0 and 1 digits. No minus (-), decimal points or separators are allowed. If you want to define a negative binary number, please use a formula like "=0-%1101".

Example: %1100101
Limits: 32 digits

Hexadecimal

Binary numbers are identified with a "#" char as the prefix. They can contain 0-9 and A-F as digits. No minus (-), decimal points or separators are allowed. If you want to define a negative hexadecimal number, please use a formula like "=0-#ABCD".

Example: #4BF80
Limits: 8 digits

Date

(check the Datetime Values chapter as well)

Dates are identified with a "@" char as the prefix, followed by the year, month and day, which have to be separated by a point (.). Years have to be between 1980 and 2047 (always four digits), months between 1 and 12 (one or two digits) and days between 1 and 31 (one or two digits).

Example: @2024.08.13
Limits: @1980.1.1 -> @2047.12.31

Time

Times are identified with a "_" char as the prefix, followed by the hour, minute and second, which have to be separated by a colon (":"). Hours have to be between 0 and 23, minutes and seconds between 0 and 59 (all one or two digits).

Example: _17:59:32
Limits: _00:00:00 -> _23:59:59

Datetime

Datetimes are a combination of a date and a time. They start with the "@" prefix and the date, which is directly followed by the "_" char and the time. For specifications and limits see above at "time" and "date".

Example: @2024.08.13_17:59:32

3410-Datetime Values


DATETIME VALUES

This sub chapter describes, how date, time and datetime values are handled in SymCalc. This information maybe useful if you want to do your own math beside using the existing datetime functions.

Internally any "date", "time" or "datetime" expression is stored in a UNIX-like way as a signed 32bit number which presents the number of seconds since 01.01.1980 00:00:00 (ten years later compared to Unix). That allows to store any date and time until 31.12.2047.

Knowing that every value is just an amount of seconds, and the length of a day is 24*60*60=86400 seconds you could now do your own calculations. E.g.

=@2024.1.1+86400*10

This is nothing else than adding 10 days to a datetime, and so in our case it will result in the 11.01.2024 .

To find out the number of weeks between two dates you could just use the following formula:

=(@2024.12.31-@2024.1.1)/86400/7

which will result in around 52,14 which proves, that one year has a little bit more than 52 weeks. Btw if you type-in this example you will probably see 1.1.1980 as a result. The reason for this is, that SymCalc is supposing that you are working with dates, and so the cell display format will be switched to "date". You have to change the format back to "number" to see the decimal value.

3500-Entering Formulas


ENTERING FORMULAS

In addition to text and numeric values, a cell can also contain a formula, which is nothing more than a mathematical expression. A formula always starts with a equality sign ("=") followed by the formula itself. This may contain numbers, cell references, operands, functions and parenthesis. SymCalc accepts formulas in the same general way you are used to in mathematics and programming.

The most simple formula is just something like
=123
which will just return the value 123.
A more useful formula would be
=A1+A2
which returns the sum of the cells A1 and A2.
Here is an advanced example, which is using parenthesis and follows the order of operations:
=(A1+A2/A3)*A4/(100-A5)

Cell references

In a formula you can use the content of other cells like variables. A cell reference consists of the letter or letters of the column and the number of the row. So A1 is just the first cell in the first row, while B4 is the second cell in the fourth row.

Cell ranges

Several functions accept lists as a parameter. Such a list can be both multiple values and cell references but also cell ranges. Cell ranges are just defined by the upper left and the lower right cell, separated by a colon (":"). As an example, the cell range B3:E4 contains the cells B3,C3,D3,E3,B4,C4,D4 and E4.

Relative and absolute references

Normally, a cell reference is treated as a relative reference. This means that the reference represents a relative distance to the current cells. An absolute cell reference is like a fixed address and is defined by writing a dollar sign ("$") before the letter and number of the reference. For example, A3 is a relative reference, while $A$3 is an absolute reference.

The difference between relative and absolute references becomes important when you copy formulas from one cell to another using the copy function. Relative references change depending on the new position of the copied cells, while absolute references always point to the same cell. Take a look at the compound interest example to get a good idea of why relative and absolute references are used.

References can also be a mixture of relative and absolute references. If only the columns are to be absolute, you just put the $ in front of the letter (e.g. $C12). If only the rows are to be absolute, you just put it in front of the number (e.g. C$12).

Order of operations

When calculating a formula, SymCalc follows the conventional order of operations. This means that operators are ranked. Those with the higher priority are executed before those with the lower priority. Parentheses always have the highest priority.

The ranking of all operators in SymCalc is as follows (from high to low):

() ^ * / + -
NAND AND XOR OR
<> = >= > <= <
&& ||

A detailed description of all operators can be found in this chapter. The order in which the operators are listed here corresponds to their precedence.

Functions

SymCalc offers a large number of functions that can be used in formulas. A function always begins with its name and an opening parenthesis, followed by parameters separated by semicolons and a closing parenthesis. Each parameter can be a numeric value, a cell reference, or even a complete formula containing other functions. This allows for complex and powerful calculations even within a single cell.

This example returns true (1) if the year specified in A1 is a leap year, otherwise it returns false (0):

=DAYS(DATE_ADD(DATE(A1;12;31);0;0;1);DATE(A1;1;1))=366

Using the mouse for cell references

When editing a formula, you can use the mouse to insert references to individual cells or cell ranges. To do this, simply click on the cell in the spreadsheet or select a cell range. Its references are then automatically inserted into the cell content.

Recalculation

Once the value of a cell has changed, SymCalc checks to see if other cells reference the updated cell. If so, those cells are recalculated and all cells that reference them are also recalculated. It doesn't matter what order the cells are in the table, SymCalc will find all references in any direction. Although this uses a particularly fast algorithm, this process can still take a while for large, more complex tables.

3520-Examples


EXAMPLES

Example 1: Compound Interest

This example is specifically intended to demonstrate the difference between relative and absolute cell references.

You want to calculate compound interest and how your money grows. To do this, enter the following into your spreadsheet:

A B C
1 Interest: 5%
2
3 Year: Amount: Owed:
4 2024 200 =B4*$C$1
5 =A4+1 =B4+C4 =B5*$C$1

Look at C4: The reference to the amount (B4) is relative, while the reference to the interest rate is absolute ($C$1). A5 (next year) and B5 (new amount) also have relative references, while C5 again contains both. Now copy the cells in row 5 and paste them into a new range from A6 to C9.

You can do this using the keyboard by navigating to cell A5, then pressing Shift and the left arrow key twice. Now press Control+C (Copy). Next, navigate to A6, press Shift and the arrow right key twice and the arrow down key twice. Now press Control+V (Paste). Or use the mouse, highlight the range from A5 to C5, right-click and select "Copy". Now highlight the range from A6 to C8, right-click and select "Paste".

You will then get the following cell content:

A B C
6 =A5+1 =B5+C5 =B6*$C$1
7 =A6+1 =B6+C6 =B7*$C$1
8 =A7+1 =B7+C7 =B8*$C$1

As you can see, all relative references have been adjusted, while the absolute references to C1 have remained the same. The result on your screen looks like this (formatted with two decimal places, using the $ unit):

A B C
1 Interest: 5%
2
3 Year: Amount: Owed:
4 2024 200,00$ 10,00$
5 2025 210,00$ 10,50$
6 2026 220,50$ 11,02$
7 2027 231,52$ 11,57$
8 2028 243,10$ 12,15$

Tip: You could now change the interest rate at C1 or the starting amount at B4 and SymCalc will automatically recalculate the entire table for you.

Additional options:

6220-Edit


"EDIT" MENU


>Cut

Prepares the current cell or the cells of a selected cell range for a cut & paste process. The cells won't be cut right now but at the moment when you start a paste process (see below).

>Copy

Copies the current cell or prepares the cells of a selected cell range for a copy & paste process (see next).

>Paste

Depending on if you did a cut or a copy before this will perform a cut & paste or a copy & paste process. Cells will be pasted either into the new specified cell range, or if no cell range is marked, they will be pasted to the area starting at the actual cell.

→ Cut & paste: The cell or selected cell range will be moved from their original area to the new area. All relative references inside the cell range will be relocated as well as all relative references outside the cell range. The size of the destination cell range always has the size of the original cell range. If you marked a new range with different sizes, this will be ignored. Cells from the source area will be cleared, but the source and the destination area can overlap each other and will be moved correctly, so no cell is lost.

→ Copy & paste: The cell or selected cell range will be pasted to the new area. If the new area is smaller than the original cell range, only the first cells will be pasted. If the new area is larger, cells will be repeated both in X and Y direction if necessary. All relative references inside the new pasted cells will be relocated.

8000-Functions


FUNCTIONS

Functions are a convenient way to perform large or complex calculations. Think of functions as a kind of short hand that simplifies computations that would be very difficult, or sometimes nearly impossible, to express as formulas.

A function may require an amount of values to produce a new value, and the number of values depends on the function. Some just require one value like SIN or SQRT, some require even three values like most financial ones, and there are also functions which don't require any value as they just return a constant like PI() or ROW(). Values are always separated by a semicolon (";").

Lists

Several functions accept a list where the number of values is not fixed to a special amount. These functions are using a parameter called "list" at the last position, and this list can be multiple parameters, cell references, cell ranges or a combination of all this.

Some examples how a list can be defined:
→ MAX(1;3;5) - contains 1, 3 and 5
→ MAX(A1:C3) - contains all cells in the range between A1 and C3
→ MAX(1;2;A1:C3) - contains 1, 2 and the cell range A1 to C3
→ MAX(1;A1:C3;C6;C7) - contains 1, the range A1 to C3 and cells C6 and C7

8500-Financial


FINANCIAL FUNCTIONS

Financial functions allow you to perform financial computations like present and future value without the use of long or complex formulas.

Most common parameters are "rate", which presents an interest or dicount rate, and "nper", which stands for "number of periods" and presents the number of payments or incomes in total or in an annuity.


PV(rate;nper;payment)
calculates the present value of a loan or an investment, based on a constant interest rate

Parameters:
rate interest rate per period
nper total number of payment
periods in an annuity
payment payment made each period and
cannot change over the life
of the annuity

Example:
PV(7%;1;1,04)
Result:
0,971962614

Formula used:
PV=payment*((1-(1+rate)^nper)/rate)


PMT(rate;nper;value)
calculates the payment for a loan based on constant payments and a constant interest rate

Parameters:
rate interest rate for the loan
nper total number of payments for
the loan
value present value, or the total
amount that a series of
future payments is worth now;
also known as the principal

Example:
PMT(15%/12;5*12;10000)
Result:
237,899299

Formula used:
PMT=value*(rate/(1-(1+rate)^(0-nper))


FV(rate;nper;payment)
calculates the future value of an investment based on a constant interest rate

Parameters:
rate interest rate per period
nper total number of payment
periods in an annuity
payment payment made each period and
cannot change over the life
of the annuity

Example:
FV(0,75%;48;100)
Result:
5752,07094

Formula used:
FV=payment*(((1+rate)^nper-1)/rate)


RATE(nper;present;future)
returns the interest rate per period of an annuity

Parameters:
nper total number of payment
periods in an annuity
present present value, the total
amount that a series of
future payments is worth now
future future value, or a cash
balance you want to attain
after the last payment is
made

Example:
RATE(3;1000;1500)
Result:
0,144714243

Formula used:
RATE=((future/present)^(1/nper))-1


NPER(rate;payment;value)
Returns the number of periods for an investment based on periodic, constant payments and a constant interest rate

Parameters:
rate interest rate per period
payment payment made each period and
cannot change over the life
of the annuity
value future value, or a cash
balance you want to attain
after the last payment is
made

Example:

Result:
11,5267046

Formula used:
NPER=
(ln(1+(value*(rate/payment))))/ln(1+rate)


NPV(rate;list)
calculates the net present value of an investment by using a discount rate and a series of future income
Parameters:
rate rate of discount over the
length of one period

each end of a period

Example:
